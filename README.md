# 药品进销存系统

#### 介绍
一个基于 **SpringBoot + Vue.js** 前后端分离架构的企业级药品进销存管理系统。本项目基于成熟的若依(RuoYi)框架进行深度二次开发，实现了药品信息、采购入库、销售出库、库存管理及数据统计等核心业务流程的信息化与自动化，有效提升了药品管理的效率和准确性。

#### 软件架构
**核心技术栈**
- **后端架构**：Spring Boot, Spring Security, MyBatis-Plus, Maven
- **数据库**：MySQL, Redis
- **前端架构**：Vue.js, Element-UI, Axios, ECharts
- **开发工具**：IDEA, WebStorm, Navicat, Git

**系统特色**
- **前后端分离**：前端Vue.js负责页面渲染与交互，后端Spring Boot提供RESTful API，耦合度低，易于维护。
- **权限控制**：基于RBAC（用户-角色-权限）模型，实现精细到按钮级别的访问控制。
- **性能优化**：集成Redis缓存，提升热点数据（如药品分类、用户信息）查询速度。

#### 安装教程

**环境准备**
1.  **JDK 1.8+**
2.  **IDEA** 或 Eclipse
3.  **MySQL 5.7+**
4.  **Redis 5+**
5.  **Maven 3.6+**
6.  **Node.js 14+**


    **本地运行步骤**
1.  **克隆代码**：
2.  **数据库初始化**：
- 创建数据库（如 `db_pss`）。
- 执行项目 `sql/` 目录下的脚本，初始化表结构及数据。
3.  **后端启动**：
- 使用IDEA打开项目，配置 `ruoyi-admin/src/main/resources/application-druid.yml` 中的数据库连接信息。
- 启动Redis服务。
- 运行 `RuoYiApplication.java` 主类，后端服务运行在 `http://localhost:8080`。
4.  **前端启动**：
- 命令行进入 `ruoyi-ui` 目录。
- 执行 `npm install` 安装依赖。
- 执行 `npm run dev` 启动前端开发服务器，访问 `http://localhost:80`。
